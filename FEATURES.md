# 🎯 功能对比与选择指南

|---------|------------|-----------|---------|
| **核心技术** | Kimi LLM + RAG | 关键词匹配 | 静态展示 |

| **追问澄清** | ✅ 支持 | ❌ | ❌ |

## 🎮 使用场景推荐

### 选择 🤖 LLM 智能版，如果你：
- 有 Kimi API Key（免费额度足够）
- 需要自然对话体验
- 问题比较复杂，需要多轮交流
- 希望获得个性化、针对性的建议
- 不介意稍微等待一下回答

**启动方式**:
```bash
cd llm_assistant
export KIMI_API_KEY=your_key
./start.sh
# 打开 frontend/chat.html
```

---

### 选择 🔧 轻量问答版，如果你：
- 没有 API Key
- 想要即开即用
- 问题比较标准、常见
- 需要快速得到答案
- 在网络不好的环境

**启动方式**:
```bash
./start.sh
# 访问 http://localhost:8080/smart.html
# 或直接双击打开 smart.html
```

---

### 选择 📚 浏览版，如果你：
- 想系统学习管理知识
- 喜欢自己翻阅查找
- 需要查看完整 Skill 内容
- 做研究或备课

**启动方式**:
```bash
./start.sh
# 访问 http://localhost:8080
```

## 📝 实际使用示例

### 场景：准备绩效面谈

**LLM 智能版**:
```
用户: 我明天要和一个绩效下滑的员工面谈，他之前表现很好，
     最近两个月突然变差，我不知道该怎么开口。

AI: 我理解您的处境。从您的描述来看，这可能是一个...
     [分析可能原因]
     
     建议的面谈策略：
     1. 先肯定过去的贡献...
     2. 用开放式问题了解变化原因...
     3. 共同制定改进计划...
     
     需要我帮您准备具体的问题清单吗？

用户: 好的，帮我准备 5 个开场问题

AI: 当然，这是 5 个建议的开场问题...
```

**轻量问答版**:
```
用户: 员工绩效突然下降怎么办

系统: 
匹配 Skills: performance-review-framework, performance-diagnosis

回答: [提取的 Skill 内容]

TODO:
🔴 收集员工绩效数据（面谈前 1 周）
🔴 准备正面反馈和改进建议（面谈前 3 天）
🟡 预约一对一会议时间
```

## 🔧 如何获取 Kimi API Key

1. 访问 [Kimi 开放平台](https://platform.moonshot.cn/)
2. 注册/登录账号
3. 进入"API Key 管理"
4. 创建新的 API Key
5. 复制并保存（只显示一次）

**免费额度**: 新用户有 15 元免费额度，足够试用

## 💡 进阶用法

### 三个版本可以一起用！

1. **日常快速查询** → 轻量问答版
2. **复杂问题深入讨论** → LLM 智能版
3. **系统学习** → 浏览版

### 团队协作

- 将轻量版或浏览版部署到内网，供团队随时查阅
- 重要的管理决策使用 LLM 版深入分析
- 导出 TODO 清单，分配给相关人员
